如何提交

  1. Fork 本仓库，新增文件到 data/submissions/，每个提交一个 JSON（见 schema 与示例）。
  2. 在 PR 中附上：评测配置（命令/迭代数/温度/系统提示等）、关键日志与结果文件（可链接至公开 gist 或 repo）。
  3. 通过自动化校验后，合并即会触发 GitHub Pages 部署，榜单自动更新。

注：指标与判定脚本应与 GitTaskBench 一致。必要时可附带复现实验的脚本与 commit 哈希。

FAQ

α 分数如何计算？

α 是兼顾效果与成本的综合分（可选）。若未提供则以 “—” 显示。建议参考论文/官方实现确定公式，并在提交中注明计算方式。

我可以只提交个别领域的结果吗？

可以。请在 domains 字段标注已覆盖的领域（如 Image/Web/Office）。排行榜默认按整体指标排序。

是否支持自动抓取 GitTaskBench 结果目录？

推荐通过 PR 方式提交聚合结果。若要自动抓取，可在 Actions 中添加脚本，从公开存储拉取 test_results 进行聚合后生成 JSON。
