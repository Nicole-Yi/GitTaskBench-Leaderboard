How to Submit

  1. Fork this repository, add files to data/submissions/, one JSON file per submission (see schema and examples).
  2. Include in PR: evaluation configuration (commands/iterations/model versions, etc.), key logs and result files (can link to public gist or repo).
  3. After automated validation, merging will trigger GitHub Pages deployment, automatically updating the leaderboard.

Note: Metrics and evaluation scripts should be consistent with the official GitTaskBench. Include reproduction scripts and commit hashes if necessary.
